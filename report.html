Predicting good exercise form from accelerometer data

Derwin McGeary

11/08/2015

Background

Using devices such as Jawbone Up, Nike FuelBand, and Fitbit it is now possible to collect a large amount of data about personal activity relatively inexpensively. These type of devices are part of the quantified self movement a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. One thing that people regularly do is quantify how much of a particular activity they do, but they rarely quantify how well they do it. In this project, we want to use data from accelerometers on the belt, forearm, arm, and dumbell of 6 participants. They were asked to perform barbell lifts correctly and incorrectly in 5 different ways. More information is available from the website here: http://groupware.les.inf.puc-rio.br/har (see the section on the Weight Lifting Exercise Dataset).

A function to strip out columns with more than 10% zeros. As computational power is limited we wish to keep only the most informative data.

philtre <- function(datas) {
  datas <- datas[sapply(datas, function(x) !any(is.na(x)))]
  
  zerotest <<- NULL
  for(i in 1:length(datas))
  {
    zerotest <<- c(zerotest, nrow(datas[datas[,i] == 0,])/nrow(datas))
  }
  removecols <- colnames(datas[,which(zerotest >.10)])
  datas[,removecols] <- list(NULL)
  datas[datas == 0] <- NA
  datas <- datas[complete.cases(datas),]
  datas
}

training<-philtre(training)

Combining models

As none of these were particularly good individually, I decided to try to increase the accuracy by using a simple voting model to combine the three. The acceleration model is the best, so we use the prediction from that unless the other two models both predict the same thing, in which case we take that value instead. This is a weighted majority-vote model.

total <- data.frame(acc=predict(accmodel,validation),gyro=predict(gyromodel,validation),dumb=predict(dumbmodel,validation),actual=validation$classe,voted=validation$classe)
total$voted <- ifelse(total$dumb==total$gyro,as.character(total$gyro),as.character(total$acc))
z <- confusionMatrix(total$actual,total$voted)
cm <- z$table
ggplot(melt(cm),
aes(Reference,
Prediction,
fill=value,
label=value)) +
geom_tile() +
geom_text() + ggtitle("Voting model")

print(estimated_accuracy <- sum(total$voted==total$actual)/length(total$voted)*100)
## [1] 95.81974

This gives an estimated accuracy on the validation set of 96 % which is an improvement on the acceleration model.
